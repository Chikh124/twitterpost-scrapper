# Як налаштувати npm в PowerShell (щоб писати просто `npm`)

## Проблема
У PowerShell команда `npm` не працює через обмеження політики виконання, тому потрібно писати `npm.cmd`.

## Рішення (2 способи)

### Спосіб 1: Автоматичне налаштування (рекомендовано)

**Подвійний клік на файл:** `fix-npm.bat`

Або в PowerShell виконайте:
```powershell
.\fix-npm.ps1
```

Після цього **перезапустіть PowerShell** і команда `npm` буде працювати!

### Спосіб 2: Вручну

1. Відкрийте PowerShell
2. Виконайте:
```powershell
notepad $PROFILE
```

3. Якщо файл не існує, PowerShell запитає чи створити - натисніть "Так"
4. Додайте в кінець файлу:
```powershell
# Функція-обгортка для npm
function npm {
    if ($args.Count -eq 0) {
        & npm.cmd
    } else {
        & npm.cmd @args
    }
}
```

5. Збережіть файл (Ctrl+S) і закрийте Notepad
6. Перезапустіть PowerShell

### Перевірка

Після налаштування перевірте:
```powershell
npm --version
```

Якщо показує версію - все працює! ✅

## Альтернатива

Якщо не хочете налаштовувати, просто використовуйте `npm.cmd` замість `npm`:
```powershell
npm.cmd install
npm.cmd start "URL"
```
